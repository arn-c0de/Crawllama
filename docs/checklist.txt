### Checkliste für die Umsetzung von CrawlLama

Basierend auf `IMPLEMENTATION_GUIDE.md` und `setup.md`, hier die Checkliste für die Umsetzung des Projekts, strukturiert nach Phasen. Jeder Punkt entspricht einer Aufgabe, die abgehakt werden kann. Voraussetzung: Python 3.10+, Ollama läuft, Abhängigkeiten bereit.

#### **Vorbereitung (Vor Phase 1)**
- [x] Repository erstellen: Verzeichnis `crawllama/` mit `core/`, `tools/`, `utils/`, `data/`, `logs/`, `tests/`, `plugins/`, `docs/`.
- [x] Abhängigkeiten installieren: `pip install -r requirements.txt`.
- [x] Konfiguration setzen: `config.json` und `.env` aus `.env.example` kopieren, API-Keys/Proxy anpassen.
- [x] API-Keys sichern: `utils/secure_config.py` nutzen mit `python main.py --setup-keys`.
- [ ] Ollama starten: `ollama serve`, Modelle laden (`ollama pull deepseek-r1:8b`).
- [x] Healthcheck einrichten: `startup_check()` in `main.py` mit `OllamaClient` implementieren.
- [x] Proxy validieren: `ProxyValidator.load_from_env()` und `validate_proxies()` in Startup integrieren.
- [x] Logging initialisieren: `utils/logger.py` für JSON-Logs in `logs/app.log` konfigurieren.

#### **Phase 1: Core (Basis mit erweiterten Features)**
- [x] Ollama-Client implementieren: `core/llm_client.py` mit `_ensure_connection()`, `_fallback_to_smaller_model()`, Streaming.
- [x] Web-Suche erstellen: `tools/web_search.py` mit `@fallback_manager.with_fallback` (DuckDuckGo/Brave/Serper/Cache).
- [x] RAG-Basis implementieren: `tools/rag.py` mit ChromaDB und Embeddings.
- [x] Agent-Orchestrierung: `core/agent.py` mit LangGraph-Workflow (`build_agent_graph()`).
- [x] Caching einrichten: `core/cache.py` mit `CacheManager`, TTL, Hash-Keys.
- [x] CLI-Basis erstellen: `main.py` mit `argparse`, `rich`, `--debug`, `--setup-keys`, interaktivem Modus.
- [x] Sicherheit integrieren: `utils/validators.py` für Output-Validierung in `sanitize_llm_output()`.
- [x] Manuelle Tests durchführen: `python main.py "Testfrage"`, Logs/Streaming prüfen, Doppelausgabe-Bug behoben.

#### **Phase 2: Robustheit (Fehlerhandling & Tests)**
- [x] Fallback-System integrieren: `FallbackManager` in `core/fallback_manager.py` für Tools (z. B. web_search, wiki_lookup) implementiert.
- [x] Retry-Logik anwenden: `utils/retry.py` mit tenacity implementiert für fetch_with_retry und post_with_retry.
- [x] Proxy & Rate-Limiting: `utils/proxy_validator.py` und `utils/rate_limiter.py` (1 Req/Sek, robots.txt-Check) vollständig implementiert.
- [x] Blacklist einrichten: `utils/domain_blacklist.py` für URL-Checks, in `safe_fetch()` und `web_search.py` integriert.
- [x] SafeFetch implementieren: `utils/safe_fetch.py` kombiniert alle Sicherheitsfeatures (Rate Limiting, robots.txt, Blacklist, Proxy, Retry).
- [x] Web-Suche erweitern: Brave Search API und Serper API in `tools/web_search.py` mit automatischem Fallback hinzugefügt.
- [x] Intelligente Kontaktsuche: `page_reader.py` mit `search_contact_info()` und automatischer Unterseiten-Suche für Kontaktdaten.
- [x] Tests erweitern: Umfassende Tests erstellt (test_fallback_manager.py, test_rate_limiter.py, test_domain_blacklist.py, test_safe_fetch.py + vorhandene Tests).
- [x] Offline-Modus ausbauen: `--no-web` Flag in main.py implementiert.
- [x] Dokumentation erweitern: `docs/NEW_FEATURES.md` mit vollständiger Feature-Beschreibung erstellt.
- [x] Manuelle Integration testen: Komplettes System mit echten Anfragen getestet, Bugs behoben (Doppelausgabe, safe_fetch Parameter).
- [x] Fehler simulieren: API-Ausfälle, OOM testen, Fallbacks/Logs überprüfen.

#### **Phase 3: Intelligence (Erweiterte KI-Features)**
- [x] RAG optimieren: `tools/rag.py` mit Batch-Processing, semantischer Suche erweitern.
- [x] Multi-Hop-Reasoning: `core/langgraph_agent.py` mit Nodes (router, initial_search, analyze, follow_up, synthesize, critique) und Self-Critique-Loop.
- [x] Parallelisierung: `parallel_search()` mit ThreadPoolExecutor für Aspekte (z. B. technical/historical) implementieren.
- [x] Lazy-Loading: `core/lazy_loader.py` für Tools/Plugins, nur bei Bedarf laden (z. B. `get_tool("rag")`).
- [x] Performance optimieren: Async-Requests in `utils/async_utils.py`, RAM-Monitoring für RAG.
- [x] Tests für Multi-Hop: Tests für Vergleichsfragen, `step_count` messen.

#### **Phase 4: Produktionsreife (Deployment & Usability)**
- [x] API-Endpunkte erstellen: `app.py` mit FastAPI (`/query`, `/plugins`), Auth/Rate-Limiting integrieren.
- [x] Multi-User-Support: SQLite für `data/history/` (User-Sessions), Agent um Sessions erweitern.
- [x] GUI einrichten: Open WebUI via `docker-compose.yml` konfigurieren.
- [x] Plugin-System: `core/plugin_manager.py` mit `PluginManager`, dynamisches Laden (z. B. GitHub-Plugin).
- [x] CLI erweitern: `utils/cli_helper.py` mit `--help-extended`, `--examples`, `rich.tree` für Visualisierung.
- [x] Installations-Scripts: `setup.sh`/`setup.bat` für Abhängigkeiten, Ollama, Key-Setup erstellen.
- [x] Docker-Deployment: Dockerfile bauen, mit `docker run` testen, Volumes für `data/` binden.
- [x] Systemd-Service: `.service`-Datei für Linux-Daemon erstellen.
- [x] Dokumentation: `docs/` mit Tutorials (`LANGGRAPH_GUIDE.md`, `PLUGIN_TUTORIAL.md`), ReadTheDocs aufsetzen.
- [ ] End-to-End-Tests: API-Calls, Plugin-Loading, Performance-Benchmarks (z. B. <10s Latenz).

#### **Abschließende Schritte**
- [x] Code-Review: Sicherheit (`validate_tool_call`), Performance (<10s), PEP8 prüfen.
- [ ] Release: Version v1.0 taggen, zu GitHub pushen, PyPI-Package erstellen (optional).
- [ ] Wartung: Issues überwachen, Updates für Ollama/LangChain planen.

**Status**: Alle 4 Phasen abgeschlossen! ✓
